## 2026-04-13 - Proper form label accessibility with useId
**Learning:** Found a missing explicit association between a `<label>` and `<input>` form element. This occurs when elements don't use nested `<label><input/></label>` wrapping, which is common in React to maintain a certain DOM structure for CSS styling. Generating stable, unique IDs using React's `useId` hook is the standard approach to resolve this and ensure accessible screen-reader behavior and click-to-focus action without ID collisions. Additionally, adding `aria-busy` to async buttons explicitly communicates loading states to screen readers.
**Action:** Always link `<label>` and `<input>` elements with `htmlFor` and `id` attributes using React's `useId()` hook when they cannot be wrapped structurally. Add `aria-busy={loading}` to async action buttons.
